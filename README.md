# RDT 3.0

我们已经完成RDT2.2，但是RDT2.2没有考虑信道上会出现**数据包丢失或者ACK丢失**的问题。

## RDT3.0

因为信道即可能产生比特错误，也可能会丢失数据包或ACK。

RDT2.x已经解决了信道可能产生比特错误的问题，但是并没有考虑信道会丢包的问题。

所以在RDT2.x的基础上，RDT3.0额外解决了数据包丢失或者ACK丢失的问题。

RDT3.0包含了RDT2.2的全部机制(CheckSum、序号、ACK)。又新增了一个倒计时计时器(Timeout Timer)：

* 发送数据包后立即启动计时器，如果在定时器超时之前收到了ACK则没出现问题；如果超时，则重传数据包。
* 处理RDT2.2的所有异常，并新增：数据包丢失、ACK反馈丢失、以及数据包/ACK过度延迟。
